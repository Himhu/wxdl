# 管理端功能重构实施清单

## 目标

以当前小程序已落地的真实业务能力为基准，补齐管理端，使管理端能够完整承接小程序业务闭环，而不是只停留在“登录 + 审批 + 配置”的轻后台状态。

---

## 一、当前小程序已落地能力

### 1. 用户与代理
- 微信登录/自动注册
- 用户资料刷新
- 代理申请
- 代理审批后身份切换
- 代理身份下的权限控制

### 2. 邀请裂变
- 邀请码生成
- 邀请海报生成
- 小程序码生成
- 我的下级/邀请记录

### 3. 数据转移
- 旧站账号密码查询余额
- 识别旧站代理身份（普通代理 / VIP代理）
- 确认转移到新站
- 旧站代理禁用链路已接通

### 4. 代理业务
- 卡密列表
- 首页统计
- 操作日志
- 积分余额

## 五、页面与接口对齐清单

### 1. 代理管理页
**目标**：后台直接管理 agent 体系，而不只是 user.role。

**需要能力**：
- 代理列表
- 代理详情
- 启用/禁用
- 余额查看
- 微信绑定标识

**后端现状**：
- 已有 agent 相关接口
- 管理端未接页面

### 2. 卡密管理页
**目标**：后台查看卡密资产和状态。

**需要能力**：
- 卡密列表
- 状态筛选
- 所属代理
- 使用状态
- 销毁记录

**后端现状**：
- 已有 cards / stats 接口
- 管理端未接页面

### 3. 充值与积分管理页
**目标**：承接小程序充值申请、积分流水、余额管理。

**需要能力**：
- 充值申请列表
- 通过/驳回
- 充值历史
- 积分流水
- 用户/代理余额查看

**后端现状**：
- agent 侧接口已有
- 管理端页面缺失
- 可能需要补管理员视角接口

### 4. 数据转移记录页
**目标**：后台审计旧站余额转移。

**需要能力**：
- 按用户查询
- 按旧站账号查询
- 查看转移金额
- 查看旧站身份
- 查看旧站禁用状态
- 查看失败原因

**后端现状**：
- 用户侧 transfer 已接通
- 管理端缺记录页
- 后端仍需补管理端查询接口

### 5. 邀请关系页
**目标**：查看谁邀请了谁，形成后台可审计的裂变关系。

**需要能力**：
- 邀请码归属
- 邀请人 / 被邀请人
- 申请状态
- 转化状态

**后端现状**：
- 用户侧 invite 已有
- 管理端视角接口不足

### 6. 操作日志页
**目标**：后台查看登录/操作日志。

**需要能力**：
- 登录日志
- 操作时间
- 操作人
- 状态
- IP

**后端现状**：
- `/api/audit/logs` 已补最小版
- 后续可扩展成完整 audit_logs

### 第二阶段：补关系与审计

7. 新增后台操作日志页
8. Dashboard 接真实统计

---

## 七、当前关键原则

1. 以小程序真实能力为准，不再做空壳后台
2. 用户体系和代理体系要分别承接清楚
3. 数据转移必须有后台审计能力
4. 管理端所有重要操作要能追溯
5. 后端已有接口优先复用，没有的再补管理员视角接口
